Типовые задачи поиска:

·       Нахождение наибольшего (наименьшего) элемента массива

·       Нахождение элемента массива, значение которого равно заданному значению

Нахождение наибольшего элемента в стопке карточек с записанными числами:

1)    Взять верхнюю карточку, записать на доске (запомнить) число как наибольшее.

2)    Взять следующую карточку, сравнить числа. Если на карточке число больше, то стереть старую запись и записать это число.

3)   
Повторить действия, описанные в пункте 2 для всех оставшихся карточек.

Программа поиска наибольшего элемента в массиве

program n_4;

var i, imax ,n: integer;

a: array of integer;

begin

n:=10;

setlength(a,n);

randomize;

for i:=0 to n-1 do a[i]:=random(100);

for i:=0 to n-1 do write (a[i],` `);

imax:=1;

for i:=0 to n-1 do

if a[i]>a[imax] then imax:=i;

write ('Наибольший элемент a[', imax, ']=', a[imax])

end.

Нахождение элемента массива с заданными свойствами

Результатом поиска элемента, значение которого равно заданному, может быть:

• n - индекс элемента массива такой, что a[n]= x, где х -

заданное число;

• сообщение о том, что искомого элемента в массиве не обнаружено.

 


Здесь:

• трём равен 4-й элемент;

• десяти равны 1-й и 9-й элементы;

• нет элемента, равного 12.

Будет найден первый из элементов, удовлетворяющих условию.

 


В случае, если массив упорядочен, то применяют другие, более

эффективные по сравнению с методом простого перебора алгоритмы,

один из которых - метод бинарного поиска.

Пусть есть упорядоченный по возрастанию массив целых чисел. Нужно

определить, содержит ли этот массив некоторое число (образец).

Метод (алгоритм) бинарного поиска реализуется следующим образом:

1. Сначала образец сравнивается со средним (по номеру) элементом массива

(рис. 5.9, а).

• Если образец равен среднему элементу, то задача решена.

• Если образец больше среднего элемента, то это значит, что искомый элемент

расположен ниже среднего элемента (между элементами с номерами sred+i и

niz) и за новое значение verh принимается sred+i, а значение niz не меняется

(рис. 5.9, б).

• Если образец меньше среднего элемента, то это значит, что искомый элемент

расположен выше среднего элемента (между элементами с номерами verh и

sred-l) и за новое значение niz принимается sred-l, а значение verh не меняется

(рис. 5.9, в).


Пример кода:

program poisk2;

const

  n = 10;

var

  a: array of integer; { массив целых }

  obrazec: integer; { образец для поиска }

  sred, verh, niz: integer; { номера среднего, верхнего и нижнего эл-тов массива }

  naiden: boolean; { признак совпадения с образцом }

  k, r: integer; { счетчик сравнений с образцом }

  i, j: integer;

begin

  setlength(a, n);

  { ввод 10 целых чисел }

  writeln('Бинарный поиск в массиве.');

  write('Введите 10 целых в одной строке через пробел и нажмите <Enter>');

  for i := 0 to n - 1 do

    read(a[i]); { здесь числа в массив введены }

 

  { Сортировка методом пузырька по возрастанию }

  writeln('Сортировка методом пузырька по возрастанию');

  for i := 0 to n - 2 do

    for j := 0 to n - 2 - i do

      if a[j] > a[j + 1] then

      begin

        k := a[j];

        a[j] := a[j + 1];

        a[j + 1] := k;

      end;

 

  { Вывод отсортированного массива }

  write('Отсортированный массив: ');

  for i := 0 to n - 1 do

    write(a[i]:4);

  writeln;

 

  { Ввод образца для поиска }

  writeln('Введите образец для поиска (целое число) ');

  readln(obrazec);

 

  { Бинарный поиск }

  verh := n - 1;

  niz := 0;

  naiden := FALSE;

  r := 0;

  writeln(' verh  niz  sred');

  repeat

    sred := (niz + verh) div 2;

    writeln(verh:5, niz:5, sred:5);

    r := r + 1;

    if a[sred] = obrazec then

      naiden := TRUE

    else

    begin

      if obrazec < a[sred] then

        verh := sred - 1

      else

        niz := sred + 1;

    end;

  until (niz > verh) or naiden;

 

  if naiden then

    writeln('Совпадение с элементом номер ', sred + 1, '. Выполнено ', r, ' сравнений.')

  else

    writeln('Образец в массиве не найден.');

 

  readln;

end.
